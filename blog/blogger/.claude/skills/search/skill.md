# Search — 블로그 리서치 스킬

블로그 포스트 작성에 필요한 자료를 웹에서 수집하고 구조화하는 스킬.

## 워크플로우

### 1. 주제 분석
사용자 입력에서 핵심 주제, 타겟 독자, 글의 목적을 파악한다.

### 2. 검색 쿼리 설계
하나의 주제에 대해 3~5개의 다른 관점의 쿼리를 생성한다:
- 정의/개요 쿼리: "{주제}란 무엇인가"
- 심화 쿼리: "{주제} 장단점", "{주제} 비교"
- 트렌드 쿼리: "{주제} 2024 트렌드", "{주제} 최신"
- 키워드 쿼리: "{주제} 관련 질문", "people also ask {주제}"
- 필요하면 영문 쿼리도 추가하여 더 넓은 소스를 확보

### 3. 검색 수행
WebSearch로 쿼리별 검색을 수행한다. 유망한 결과는 WebFetch로 상세 내용을 수집한다.

**소스 다양성 확보:**
- 공식 문서/블로그 (권위성)
- 뉴스/미디어 (시의성)
- 전문 블로그/포럼 (실무 관점)
- 통계/보고서 (데이터)

### 4. 정보 구조화
수집한 정보를 리서치 보고서 형식으로 정리한다. 모든 정보에 출처 URL을 기록한다.

**보고서 구조:**
```
# 리서치 결과: {주제}

## 핵심 요약
[3~5문장]

## 주요 발견
### 1. [소주제]
- 내용
- 출처: [URL]

## 통계 및 데이터
- [수치]

## 키워드 및 트렌드
- 주요 키워드: [목록]
- 연관 질문: [목록]
- 트렌드: [동향]

## 참고 자료
1. [제목](URL) — 요약
```

### 5. 출력
`_workspace/01_searcher_results.md`에 저장한다.

## 품질 기준
- 최소 5개 이상 서로 다른 소스
- 모든 사실에 출처 URL 포함
- 상충 정보는 삭제하지 않고 출처 병기
- 키워드 섹션에 최소 5개 이상의 키워드 후보
