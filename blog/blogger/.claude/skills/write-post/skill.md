# Write Post — 블로그 포스트 작성 스킬

리서치 결과를 독자에게 가치를 제공하는 블로그 포스트로 변환하는 스킬.

## 워크플로우

### 1. 리서치 결과 분석
`_workspace/01_searcher_results.md`를 읽고 핵심 메시지, 핵심 데이터, 독자 관심 포인트를 파악한다.

### 2. 포스트 구조 설계
리서치에서 추출한 소주제들을 논리적 흐름으로 배치한다:

```
도입부 → 배경/맥락 → 핵심 내용 (2~4개 섹션) → 실용적 조언 → 결론
```

구조 설계 원칙:
- 독자가 가장 궁금해할 내용을 앞쪽에 배치
- 각 섹션이 자연스럽게 다음 섹션으로 이어지도록 전환 설계
- 하나의 섹션 = 하나의 핵심 메시지

### 3. 포스트 작성

**도입부 (첫 2~3문장이 핵심):**
- 독자의 문제/궁금증을 직접 언급하여 공감 유도
- 이 글을 읽으면 얻을 수 있는 가치를 명확히 제시
- 질문, 통계, 일화 등으로 주의를 끌기

**본문:**
- 정보를 나열하지 말고 스토리로 엮는다
- 추상적 설명 후에는 반드시 구체적 예시나 데이터를 제공
- 목록, 표, 코드 블록 등 시각적 요소를 적극 활용
- 전문 용어에는 간단한 설명을 곁들인다
- 출처가 있는 사실에는 인라인 링크를 포함

**결론:**
- 핵심 내용을 2~3문장으로 요약
- 독자가 다음에 할 수 있는 구체적 행동을 제안

### 4. 메타 데이터 작성
frontmatter에 title, description, date, tags를 포함한다.

### 5. 출력
`_workspace/02_post-writer_draft.md`에 저장한다.

## 글쓰기 원칙
- 한 문단은 3~5문장을 넘지 않는다. 긴 문단은 독자를 지치게 한다.
- 능동태를 사용한다. "~되었다" 보다 "~했다".
- 불필요한 수식어를 제거한다. "매우", "정말", "엄청" 등은 대부분 삭제해도 의미가 유지된다.
- H2는 5~7개, H3는 각 H2 아래 2~3개가 적정.
